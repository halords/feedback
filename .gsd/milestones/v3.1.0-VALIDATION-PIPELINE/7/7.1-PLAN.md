# Plan 7.1: User Management Update

## Objective
Enhance the user management service and API to support toggling the `analyticsEnabled` flag (stored as `is_analytics_enabled` on Firestore `user_data`). Add UI mechanism to allow Superadmins to change this flag for existing users.

## Context
- .gsd/SPEC.md
- .gsd/FINALrev.md
- src/lib/services/userService.ts
- src/app/api/users/route.ts
- src/app/users/UsersClient.tsx

## Tasks

<task type="auto">
  <name>Extend UserService</name>
  <files>src/lib/services/userService.ts</files>
  <action>
    Add a new exported async function `updateUserAnalyticsFlag(idno: string, isEnabled: boolean)` that:
    1. Queries the `user_data` collection by `idnumber == idno`.
    2. Updates the `is_analytics_enabled` flag (boolean) on the found document.
    3. Ensure error handling is robust.
  </action>
  <verify>grep_search export function updateUserAnalyticsFlag in src/lib/services/userService.ts</verify>
