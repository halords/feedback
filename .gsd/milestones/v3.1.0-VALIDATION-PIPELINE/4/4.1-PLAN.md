# Plan 4.1: Password Override Mechanism

## Objective
Implement a password-protected override system to allow Superadmins to modify data for archived months.

## Context
- .gsd/SPEC.md
- .gsd/phases/4/RESEARCH.md
- src/components/physical-reports/PhysicalReportsEditor.tsx
- src/components/layout/Shell.tsx

## Tasks

<task type="auto">
  <name>Implement Password Challenge Modal</name>
  <files>src/components/ui/ArchiveOverrideModal.tsx</files>
  <action>
    Create a reusable modal for the password challenge:
    - Input for the override password.
    - Validate against a hardcoded value for now (e.g., 'ADMIN-GS-2024') or use an environment variable.
    - On success, set a state/context flag `isOverrideActive`.
  </action>
  <verify>Open the modal, enter the correct password, and confirm success state.</verify>
  <done>Reusable password challenge component is ready.</done>
</task>

<task type="auto">
  <name>Integrate Override into Physical Reports</name>
  <files>src/components/physical-reports/PhysicalReportsEditor.tsx</files>
  <action>
    Update the Physical Reports Editor to respect the override:
    1. If a month is archived, but `isOverrideActive` is true, allow "Edit" and "Add" actions.
    2. Change the warning banner to a "Warning: Override Mode Active" message.
    3. Update the `View` button back to `Edit` if override is active.
  </action>
  <verify>Access an archived month, activate override, and confirm editing is unlocked.</verify>
  <done>Superadmins can modify archived physical reports in override mode.</done>
</task>

## Success Criteria
- [ ] Archived months are read-only by default.
- [ ] Correct password entry unlocks editing capabilities.
- [ ] Clear UI indicators show when Override Mode is active.
