## Lessons Learned
- **Vitest Advantage**: Chose Vitest over Jest for zero-config Next.js/TS compatibility and speed. It proved highly effective for isolated unit testing of security logic.
- **Strict Validation**: Moving from loose `request.json()` destructuring to Zod validation immediately flushed out minor inconsistencies in expected frontend payloads (e.g., `user_type` data types). This hardening pattern is now the gold standard for the repository.
- **Doc-First Security**: Formally documenting the `firestore.rules` constraint in `ARCHITECTURE.md` helps prevent future regression by reminding developers that client-side Firestore listeners are explicitly blocked.
