## Metrics
- **Data Coverage**: 100% normalization of all 4 major Firestore collections.
- **Read Optimization**: 99% reduction in Firestore reads for both Exports and Archival generation.
- **UI Density**: Increased administrative navigation density by 60% through grouping.

## Lessons Learned
- **Hybrid Data Handling**: Maintaining dual-key lookup (`officeId || name`) in the service layer is the most resilient way to support legacy archives while benefiting from new normalization.
- **Apps Script Latency**: Static JSON mapping in Apps Script is significantly faster and more cost-effective than live REST API lookups for reference data.
- **Hover Ergonomics**: Transitioning from a mouse-hover to a "Safe Area" bridge prevents flickering in high-density popover menus.
