## Concerns
- **Backend Infrastructure Defense**: Code-level restrictions are strong, but deep Firestore document-level schema security rules aren’t continuously tested.
- **Node Environment**: Though warnings were resolved via `package.json`, production server requires actual Node 20+ runtime context to fully validate Next.js 15+ engine rules.

## Recommendations
1. **Automated RBAC Execution Pipeline**: Setup mock user contexts and rigorously query data APIs on CI pipeline. 
2. **Expansion to Rules**: Map all Zod validation logic back to Firebase security rules for complete double-authorization boundary.
