## Lessons Learned
- **Metadata Latency**: Firebase custom claims take time to propagate to the session cookie. A Firestore-backed fallback in 'verifySession.ts' is essential for a seamless "First Login" experience.
- **Identifier Handling**: Allowing both Username and ID Number for login required a robust resolution layer in the Login flow to maintain a single source of truth in Firebase Auth.
